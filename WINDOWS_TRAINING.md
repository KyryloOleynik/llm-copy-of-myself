# Windows CUDA training

Use native 64-bit Windows, an NVIDIA driver compatible with the installed
PyTorch CUDA wheel, and Python 3.11. Keep the repository on an NTFS path with
enough space for the base model and checkpoints.

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements-windows-cuda.txt
python -c "import torch; print(torch.__version__, torch.version.cuda, torch.cuda.get_device_name(0))"
```

The requirements file installs the CUDA 12.8 PyTorch wheel plus Transformers,
Datasets, Accelerate, PEFT, TRL, bitsandbytes, safetensors, NumPy, and the local
project. If the target NVIDIA driver cannot support this wheel, update the CUDA
index URL using the command generated by PyTorch's official installer page.

Prepare the current intermediate dataset and run the bounded smoke test:

```powershell
personal-ai prepare-data
personal-ai train --smoke
```

Run the full initial experiment:

```powershell
personal-ai train
```

Resume a checkpoint explicitly:

```powershell
personal-ai train --resume artifacts/training/qwen3-8b-r16/checkpoint-100
```

The trainer refuses to run without CUDA or BF16 support. It uses 4-bit NF4,
double quantization, BF16 compute, the LoRA modules and hyperparameters from
`PLAN.md`, and masks every token except the final assistant reply. Watch VRAM
with `nvidia-smi`; the 12 GB acceptance limit must be verified on the target PC.

## Important data gate

`prepare-data` currently splits the existing `dataset.json` so the training
path can be tested. That file has not yet passed the plan's full redaction,
deduplication, and audit requirements. Do not perform a real personal-data
training run until those stages are implemented and manually reviewed.
