# Windows CUDA training

Use native 64-bit Windows, an NVIDIA driver compatible with the installed
PyTorch CUDA wheel, and Python 3.11. Keep the repository on an NTFS path with
enough space for the base model and checkpoints.

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements-windows-cuda.txt
python -c "import torch; assert torch.cuda.is_available(), 'CUDA-enabled PyTorch was not installed'; print(torch.__version__, torch.version.cuda, torch.cuda.get_device_name(0))"
```

The requirements file installs the CUDA 12.8 PyTorch wheel plus Transformers,
Datasets, Accelerate, PEFT, TRL, bitsandbytes, safetensors, NumPy, and the local
project. If the target NVIDIA driver cannot support
this wheel, update the CUDA index URL using the command generated by PyTorch's
official installer page.
The CUDA-qualified PyTorch version is intentionally pinned: allowing an
unqualified newer version lets pip choose a CPU-only wheel from PyPI.

Prepare the tokenizer-budgeted session-isolated dataset and run the bounded smoke test:

```powershell
personal-ai prepare-data
personal-ai train --smoke --fresh
```

The smoke test runs one worst-case optimizer step without evaluation or periodic
checkpointing. Full validation uses a per-device batch size of one because Qwen3's
full-vocabulary loss can otherwise require several additional GiB of VRAM.

Every smoke run also writes
`artifacts/training/qwen3-4b-instruct-2507-r16/smoke-samples.json`.
It lists the 20 longest selected examples, the exact examples actually collated into
the optimizer step, the prompt messages masked out of the loss, and the final assistant
reply that the model was trained to generate. Its `input_ids` and `training_labels` are
captured inside the exact same collator call that feeds Trainer, rather than being
reconstructed later by separate audit logic. Samples are moved to
`actually_used_for_backward` only after Trainer completes backward successfully;
Accelerate's one-batch look-ahead is listed separately and is not reported as trained.
This file contains unredacted private conversation text and must remain private.

Qwen3-4B-Instruct-2507 uses the standard text-only Transformers causal-LM path with
PyTorch SDPA; it does not require Qwen3.5 DeltaNet, FLA, or vision components. Gradient
checkpointing intentionally recomputes activations to reduce memory use.

Run the full initial experiment:

```powershell
personal-ai train
```

This automatically resumes the newest checkpoint in the configured training
directory when one exists and refuses to resume if its dataset hash differs.
To delete existing full-run checkpoints/adapter and start a new adapter from
the base model while preserving the matching smoke-test gate, use:

```powershell
personal-ai train --fresh
```

Resume a checkpoint explicitly:

```powershell
personal-ai train --resume artifacts/training/qwen3-4b-instruct-2507-r16/checkpoint-100
```

Or automatically resume the newest checkpoint:

```powershell
personal-ai train --resume last
```

The default configuration evaluates and saves every 50 optimizer steps and keeps the newest
two periodic checkpoints. Pressing Ctrl+C also writes a resumable
checkpoint before exiting; wait for the "Interrupted safely" message before
closing the terminal. The interrupt checkpoint is first written to a staging
directory and verified (adapter weights, trainer state, optimizer, and
scheduler) before it is promoted to `checkpoint-N`, so a partial save will not
be selected for automatic resume.

To discard all trained adapters and checkpoints while retaining the downloaded
base `Qwen/Qwen3-4B-Instruct-2507` model in the Hugging Face cache, run from the repository
root:

```powershell
Remove-Item -LiteralPath .\artifacts\training\qwen3-4b-instruct-2507-r16 -Recurse -Force
```

After deleting the directory, rerun the smoke test because its dataset gate was
also removed. `--fresh` performs the safer routine cleanup automatically and
prevents checkpoints from different prepared-dataset hashes from being mixed.

The trainer refuses to run without CUDA or BF16 support. It uses 4-bit NF4,
double quantization, BF16 compute, Qwen3 attention plus MLP LoRA modules,
and masks every token except the final assistant reply. Samples over the configured
sequence length discard old prompt context while preserving the relationship/system
prompt when possible and retaining the newest prompt tail. The complete final assistant
reply is always preserved, and a target that cannot fit is rejected.
The checkpoint is text-only. Watch VRAM with `nvidia-smi`; the 12 GiB acceptance
limit must be verified on the target PC and is recorded in `reproducibility.json`.

## Important data gate

`prepare-data` intentionally preserves unredacted private data, including credentials,
as requested. The raw export, generated datasets, adapters, checkpoints, logs, and
indexes must remain private. Do not upload any generated training artifact to a public
repository or model registry.

After training, run `personal-ai evaluate`. Complete and save the generated
`blind_style_review.json`, then rerun evaluation; matching choices are counted
automatically. Do not migrate the bot unless the selected adapter is marked accepted.
